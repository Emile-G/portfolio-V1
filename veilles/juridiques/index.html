    Présentation & Création d'un premier projet Laravel.

    Tout d'abord, qu'est-ce qu'un framework PHP. Un framework PHP est un cadre structurant une application. En général un framework permettra de répondre facilement aux problèmes souvent rencontrés par un développeur tout en accelerant le développement général du projet

    Cependant, avant de ce lancer à coeur ouvert dans un framework, il est recommandé de posséder des compétences dans le language sur le quelle il est basé (ici en PHP orienté objet), afin d'avoir la logique de la programmation, ainsi que les compétences nécessaires afin de comprendre réellement ce que vous faites.

    Il y a quelque prérequis niveau au niveau de votre machine/serveur pour pouvoir générer un projet Laravel 7.

    Pour pouvoir générer votre projet, votre machine nécessite PHP 7.2.5 au minimum, les extensions PHP suivante : BCMath, Ctype Fileinfo, JSON, Mbstring, OpenSSL, PDO, Tokenizer, XML.

    Afin de générer votre projet, Laravel passe par "Composer". Composer est un outil de gestion des dépendances en PHP. Il vous permet de déclarer les bibliothèques dont votre projet dépend et il les gérera pour vous.

    Une fois que tous ces prérequis sont installés et disponibles sur votre machine, il ne vous reste plus qu'a exécuté la commande suivante dans un terminal et dans le répertoire que vous souhaitez générer votre projet :

    composer create-project --prefer-dist laravel/laravel nom_du_projet

    Cependant, si vous souhaitez générer votre projet dans une version bien spécifique de laravel, vous avez juste à suivre cette commande :

    composer create-project --prefer-dist laravel/laravel nom_du_projet "6.*"

    Cette commande vous génèrera un projet laravel en version 6.x .

    Pour pouvoir vérifier que votre projet est bel est bien généré vous avez plus cas vous rendre dans votre répertoire via une invite de commande, et d'exécuter la commande suivante :

    php artisan serve

    Ce qui vous permettra de vous rendre sur votre projet en version de développement via l'adresse suivante par défaut :

    http://localhost:8000

    Normalement, à la racine de votre projet vous devriez trouver un fichier nommer ". env". Ce fichier contient quelque ligne vous permettant, entre autres, de configurer l'accès du projet à une base de données.
